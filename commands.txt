# Команды для просмотра логов в реальном времени (Real-time monitoring)

Все команды нужно выполнять в терминале (PowerShell), находясь в главной папке проекта:
Каталог: C:\Diplom\Web

=== 1. Сервер (Бэкенд / API / WebSockets) ===
Самое важное — видеть, как приходят запросы от веб-сайта, мобильного приложения и железа (ESP32).

Команда: 
docker compose logs -f server

(Флаг -f означает "follow", то есть терминал останется открытым и будет выводить новые строки в реальном времени, как только кто-то нажмет кнопку или отправит данные. Чтобы выйти из режима просмотра, нажмите Ctrl+C).


=== 2. Смартфон (Мобильное приложение APK) ===
Если вы скачали APK на телефон (Release build) и вам нужно понять, почему что-то не работает или не нажимается:
Вам нужно подключить телефон кабелем к ПК, включить "Отладку по USB" (USB Debugging) в настройках разработчика на телефоне.

Команда в PowerShell:
adb logcat | Select-String "flutter"

(Это выведет все ошибки и принты именно от Flutter-приложения, запущенного на вашем реальном смартфоне. Если у вас не установлен adb, его можно скачать из Android SDK Platform-Tools).


=== 3. База данных (MongoDB) ===
Если нужно отследить, что именно пишется в базу или происходят ли какие-то системные сбои в БД:

Команда:
docker compose logs -f mongo


=== 4. MQTT-брокер (Общение с железом/ESP32) ===
Чтобы следить за тем, как ESP32 подключается и публикует бронирования/статусы, можно смотреть логи брокера:

Команда:
docker compose logs -f mqtt


=== 5. Общие команды логов Docker ===
Посмотреть логи ВСЕХ контейнеров одновременно (сервер, базы, веб, мобилка):
docker compose logs -f

Посмотреть только последние 50 строк логов сервера (полезно, если лог слишком длинный):
docker compose logs -f --tail 50 server


=== 6. Тестирование веб-интерфейса ===
Логи браузера для веб-приложения (`localhost:8080` или `localhost:3000`) недоступны в терминале напрямую (контейнер отдает статику через nginx). 
Чтобы отлаживать веб в реальном времени:
1. Откройте сайт в браузере.
2. Нажмите F12 (откроются инструменты разработчика).
3. Перейдите во вкладку "Console" (Консоль) — там видны все ошибки (красные) и загрузки.
4. Перейдите во вкладку "Network" (Сеть) — там видно, какие запросы уходят на сервер при нажатии кнопок.

=== 7. Управление сервером (Docker Compose) ===
Если вы запускаете проект на ноутбуке (или после перезагрузки), вам понадобятся эти команды (выполнять в папке Web):

Запуск всей системы (бэкенд, база данных, веб-сайт, mqtt):
docker compose up -d

Остановка всей системы (чтобы не ело ресурсы, когда не нужно):
docker compose stop

Остановка с полным удалением контейнеров (данные БД сохранятся в volumes):
docker compose down

Пересборка проекта (ОБЯЗАТЕЛЬНАЯ команда, если вы поменяли код в папке server или web):
docker compose up -d --build
